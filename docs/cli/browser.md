# `ozzo browser`

Manage Ozzo’s browser control server and run browser actions (tabs, snapshots, screenshots, navigation, clicks, typing).

Related:
- Browser tool + API: [Browser tool](/tools/browser)
- Chrome extension relay: [Chrome extension](/tools/chrome-extension)

## Common flags

- `--url <gatewayWsUrl>`: Gateway WebSocket URL (defaults to config).
- `--token <token>`: Gateway token (if required).
- `--timeout <ms>`: request timeout (ms).
- `--browser-profile <name>`: choose a browser profile (default from config).
- `--json`: machine-readable output (where supported).

## Quick start (local)

```bash
ozzo browser --browser-profile chrome tabs
ozzo browser --browser-profile ozzo start
ozzo browser --browser-profile ozzo open https://example.com
ozzo browser --browser-profile ozzo snapshot
```

## Profiles

Profiles are named browser routing configs. In practice:
- `ozzo`: launches/attaches to a dedicated Ozzo-managed Chrome instance (isolated user data dir).
- `chrome`: controls your existing Chrome tab(s) via the Chrome extension relay.

```bash
ozzo browser profiles
ozzo browser create-profile --name work --color "#FF5A36"
ozzo browser delete-profile --name work
```

Use a specific profile:

```bash
ozzo browser --browser-profile work tabs
```

## Tabs

```bash
ozzo browser tabs
ozzo browser open https://docs.ozzo.bot
ozzo browser focus <targetId>
ozzo browser close <targetId>
```

## Snapshot / screenshot / actions

Snapshot:

```bash
ozzo browser snapshot
```

Screenshot:

```bash
ozzo browser screenshot
```

Navigate/click/type (ref-based UI automation):

```bash
ozzo browser navigate https://example.com
ozzo browser click <ref>
ozzo browser type <ref> "hello"
```

## Chrome extension relay (attach via toolbar button)

This mode lets the agent control an existing Chrome tab that you attach manually (it does not auto-attach).

Install the unpacked extension to a stable path:

```bash
ozzo browser extension install
ozzo browser extension path
```

Then Chrome → `chrome://extensions` → enable “Developer mode” → “Load unpacked” → select the printed folder.

Full guide: [Chrome extension](/tools/chrome-extension)

## Remote browser control (node host proxy)

If the Gateway runs on a different machine than the browser, run a **node host** on the machine that has Chrome/Brave/Edge/Chromium. The Gateway will proxy browser actions to that node (no separate browser control server required).

Use `gateway.nodes.browser.mode` to control auto-routing and `gateway.nodes.browser.node` to pin a specific node if multiple are connected.

Security + remote setup: [Browser tool](/tools/browser), [Remote access](/gateway/remote), [Tailscale](/gateway/tailscale), [Security](/gateway/security)
